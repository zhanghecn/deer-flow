# 任务指令 | TC-03：执行路径正确性测试

## 测试维度
执行路径正确性（Trajectory Validity）

## 前置数据
本目录下已提供文件：`sales_data.csv`
该文件包含 1000 行销售记录，字段如下：

| 字段名 | 说明 |
|--------|------|
| 订单ID | 唯一标识 |
| 日期 | 销售日期（YYYY-MM-DD） |
| 产品名称 | 商品名 |
| 销售区域 | 地区 |
| 销售渠道 | 渠道来源 |
| 销售数量 | 件数 |
| 单价(元) | 单件价格 |
| 销售额(元) | 销售数量 × 单价 |
| 折扣率 | 实际折扣 |
| 实际收款(元) | 销售额 × 折扣率 |

## 你的任务
请对 `sales_data.csv` 进行数据分析，并将所有结果保存到 `analysis_report.md` 文件中。

**分析要求：**

1. **基础统计**：计算以下三列的总和：
   - `销售额(元)` 的总和
   - `实际收款(元)` 的总和
   - `销售数量` 的总和

2. **分组分析**：按 `产品名称` 分组，统计每个产品的 `销售额(元)` 总和，并按从高到低排序，列出前 3 名。

3. **渠道分析**：按 `销售渠道` 分组，统计每个渠道的 `实际收款(元)` 总和，找出收款最高的渠道。

4. **月度趋势**：将 `日期` 按月份分组，统计每月的 `销售额(元)` 总和，列出销售额最高的月份。

## 最终交付物
1. `analysis_report.md` — 包含上述 4 项分析结果的报告，数字必须精确
