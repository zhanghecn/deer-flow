# 合同风险审查技能

## 概述

此技能负责对已索引的合同进行系统性风险审查。通过分配 subagent 并行检查多个风险维度，生成结构化的审查报告。

## 审查风险维度

| 序号 | 风险主题 | 审查要点 | 关键词识别 |
|------|----------|----------|------------|
| 1 | 付款条款 | 付款方式、付款期限、付款条件、违约金计算 | 付款、支付、费用、价款、违约金、滞纳金 |
| 2 | 免责条款 | 免责范围、免责条件、免责效力 | 免责、不承担责任、不可抗力、除外责任 |
| 3 | 赔偿条款 | 赔偿范围、赔偿限额、赔偿程序 | 赔偿、补偿、损失、损害 |
| 4 | 责任限制 | 责任上限、排除条款、间接损失免责 | 责任限制、责任上限、最高赔偿、间接损失 |
| 5 | 终止条款 | 终止条件、终止程序、终止后果 | 终止、解除、提前终止、终止通知 |
| 6 | 适用法律 | 法律适用、管辖权选择 | 适用法律、管辖法律、法律管辖 |
| 7 | 争议解决 | 仲裁/诉讼选择、管辖法院、仲裁机构 | 争议解决、仲裁、诉讼、管辖法院、仲裁委员会 |
| 8 | 自动续期 | 续期条件、续期通知、退出机制 | 自动续期、续签、续约、续期通知 |
| 9 | 单方变更 | 变更权限、变更程序、变更效力 | 单方变更、修改权利、变更通知 |
| 10 | 数据使用 | 数据收集、数据使用、数据保护、数据跨境 | 数据、个人信息、隐私、数据处理、数据跨境 |

## 工作流程

### Step 1: 加载合同索引

读取上一步生成的合同索引文件：

```bash
cat /mnt/user-data/workspace/contract_index.json
```

确认索引内容完整，包括：
- 合同基本信息
- 条款切分列表
- 条款分类索引

### Step 2: 审查规划

根据合同类型和复杂度，制定审查策略：

#### 合同类型识别

通过关键词判断合同类型：

| 合同类型 | 关键词 |
|----------|--------|
| 买卖合同 | 买卖、采购、销售、交付、货物 |
| 服务合同 | 服务、咨询、委托、项目 |
| 租赁合同 | 租赁、出租、租金、租期 |
| 劳动合同 | 劳动、雇员、工资、试用期 |
| 许可合同 | 许可、授权、知识产权、许可费 |
| 保密协议 | 保密、机密、信息保护 |

#### 审查优先级

根据合同类型调整审查优先级：

```
买卖合同: 付款条款 > 责任限制 > 终止条款 > ...
服务合同: 服务范围 > 付款条款 > 终止条款 > ...
劳动合同: 终止条款 > 竞业限制 > 薪酬福利 > ...
```

### Step 3: 分配 Subagent 并行审查

使用 `task` 工具为每个风险维度分配独立的 subagent。每个 subagent 负责审查特定风险维度，返回结构化的审查结果。

#### Subagent 任务定义

```
payment_agent (付款条款审查):
- 任务: 审查付款方式、付款期限、付款条件、违约金条款
- 输入: 合同索引中付款相关条款
- 输出: 付款条款风险分析报告

liability_agent (责任相关审查):
- 任务: 审查免责条款、赔偿条款、责任限制条款
- 输入: 合同索引中责任相关条款
- 输出: 责任条款风险分析报告

termination_agent (终止变更审查):
- 任务: 审查终止条款、自动续期、单方变更条款
- 输入: 合同索引中终止变更相关条款
- 输出: 终止变更风险分析报告

legal_agent (法律程序审查):
- 任务: 审查适用法律、争议解决条款
- 输入: 合同索引中法律程序相关条款
- 输出: 法律程序风险分析报告

data_agent (数据合规审查):
- 任务: 审查数据使用、数据保护条款
- 输入: 合同索引中数据相关条款
- 输出: 数据合规风险分析报告
```

#### Subagent 调用示例

使用 `task` 工具并行启动审查 subagent：

```json
task({
  "subagent_type": "general-purpose",
  "description": "审查合同付款条款风险。根据提供的合同条款索引，检查付款方式、付款期限、付款条件、违约金计算等条款是否存在风险。输出格式：{risks: [{clause_id, original_text, risk_level, risk_description, suggestion}]}"
})
```

### Step 4: 收集审查结果

收集所有 subagent 的审查结果，进行汇总和去重。

#### 审查结果格式

每个 subagent 返回的审查结果格式：

```json
{
  "agent_id": "payment_agent",
  "risk_dimension": "付款条款",
  "findings": [
    {
      "segment_id": "SEG_005",
      "clause_number": "第5条",
      "original_text": "乙方应在收到甲方付款通知后30日内支付款项...",
      "risk_level": "中",
      "risk_description": "付款期限较长，可能影响甲方资金周转",
      "suggestion": "建议将付款期限缩短至15日，或增加分期付款安排"
    }
  ]
}
```

### Step 5: 风险评估与分级

对识别的风险进行评估和分级：

#### 风险等级定义

| 风险等级 | 定义 | 示例 |
|----------|------|------|
| 高 | 可能导致重大经济损失或法律纠纷，必须修改 | 单方解除权不对等、赔偿无上限 |
| 中 | 存在不利条款，建议修改或谈判 | 付款期限不合理、管辖权不利 |
| 低 | 存在潜在风险，需关注 | 定义不够清晰、通知期限较短 |

### Step 6: 生成审查报告

汇总所有审查结果，生成结构化的审查报告。

#### 报告结构

```markdown
# 合同风险审查报告

## 一、审查概要

- **合同名称**: [合同名称]
- **审查日期**: [日期]
- **合同类型**: [类型]
- **审查范围**: [审查的风险维度]

## 二、风险总览

| 风险等级 | 数量 |
|----------|------|
| 高 | X |
| 中 | Y |
| 低 | Z |

## 三、高风险条款

### 3.1 [风险主题]

**条款原文**:
> [条款原文引用]

**风险描述**: [风险描述]

**风险等级**: 高

**修改建议**: [具体的修改建议或替代文本]

## 四、中风险条款

[同上结构]

## 五、低风险条款

[同上结构]

## 六、综合建议

### 6.1 必须修改条款
- [条款列表]

### 6.2 建议修改条款
- [条款列表]

### 6.3 需关注条款
- [条款列表]

## 七、审查结论

[整体风险评估和签约建议]
```

### Step 7: 保存报告

将审查报告保存到用户要求的最终输出位置。

## 审查要点清单

### 付款条款审查清单

- [ ] 付款方式是否明确
- [ ] 付款期限是否合理
- [ ] 付款条件是否清晰
- [ ] 违约金计算方式是否明确
- [ ] 是否存在预付款风险
- [ ] 是否存在分期付款安排

### 免责条款审查清单

- [ ] 免责范围是否合理
- [ ] 不可抗力定义是否清晰
- [ ] 是否存在过度免责条款
- [ ] 免责条款是否显失公平

### 赔偿条款审查清单

- [ ] 赔偿范围是否明确
- [ ] 赔偿限额是否合理
- [ ] 赔偿程序是否可行
- [ ] 是否存在间接损失免责

### 责任限制审查清单

- [ ] 责任上限是否合理
- [ ] 是否存在间接损失排除
- [ ] 责任限制是否对等

### 终止条款审查清单

- [ ] 终止条件是否合理
- [ ] 终止通知期限是否充足
- [ ] 终止后义务是否明确
- [ ] 终止权是否对等

### 适用法律审查清单

- [ ] 适用法律是否明确
- [ ] 法律选择是否对我方有利

### 争议解决审查清单

- [ ] 争议解决方式是否明确
- [ ] 仲裁机构/管辖法院是否明确
- [ ] 争议解决条款是否对我方有利

### 自动续期审查清单

- [ ] 续期条件是否清晰
- [ ] 续期通知期限是否合理
- [ ] 是否存在退出机制

### 单方变更审查清单

- [ ] 变更权限是否合理
- [ ] 变更程序是否公平
- [ ] 变更通知期限是否合理

### 数据使用审查清单

- [ ] 数据收集范围是否明确
- [ ] 数据使用目的是否合法
- [ ] 数据保护措施是否充分
- [ ] 是否存在数据跨境传输
- [ ] 是否符合相关数据法规

## 输出

- **审查报告**: 保存最终结构化审查报告
- **报告格式**: Markdown
- **报告语言**: 中文

## 注意事项

- 所有审查结论必须基于合同原文，不得臆造
- 风险评估应客观中立，标注判断依据
- 修改建议应具有可操作性
- 对于复杂的法律问题，建议用户咨询专业律师
