# SaaS Landing Page Copywriter Skill

## Overview

This skill provides proven frameworks, formulas, and best practices for writing high-converting SaaS landing page copy.

## When to Use

Use this skill when:
- Writing hero section headlines and subheadlines
- Crafting feature descriptions and benefit statements
- Creating calls-to-action (CTAs)
- Developing social proof elements (testimonials, case studies)
- Building objection handlers and urgency builders

## Core Frameworks

### 1. Hero Section Formula

**Structure:**
```
Headline: [Outcome] + [Timeframe/Method] + [Differentiator]
Subheadline: [Problem Agitation] → [Solution Promise] → [Social Proof Hint]
CTA: [Action Verb] + [Benefit] + [Risk Reversal]
```

**Examples:**
- "Close 3x More Deals in Half the Time with AI-Powered Sales Intelligence"
- "Stop Losing Customers to Slow Support. Resolve Tickets 10x Faster with Smart Automation."

### 2. Feature-to-Benefit Translation

**Formula:**
```
[Feature] → [Direct Benefit] → [Emotional Outcome]
```

**Template:**
- Instead of: "Real-time analytics dashboard"
- Write: "Watch your metrics update instantly (Feature) so you can spot trends before competitors (Benefit) and feel confident in every decision (Emotional Outcome)."

### 3. PAS Framework (Problem-Agitate-Solution)

**Structure:**
1. **Problem**: Identify the pain point
2. **Agitate**: Amplify the consequences of not solving it
3. **Solution**: Present your product as the answer

**Example:**
- Problem: "Your team wastes hours on manual data entry every week."
- Agitate: "That means missed deadlines, burnt-out employees, and revenue slipping through the cracks."
- Solution: "Our automation platform eliminates 90% of data entry tasks—freeing your team to focus on high-value work."

### 4. CTA Best Practices

**High-Converting CTA Formulas:**
1. "Start [Benefit] — [Timeframe]" → "Start Saving 10 Hours/Week — Free for 14 Days"
2. "[Action] + [Outcome] + [No-Risk]" → "Get Instant Access — No Credit Card Required"
3. "[Action] + [Specific Benefit]" → "See Your SEO Score Instantly"

**CTA Button Copy Rules:**
- Use action verbs (Get, Start, See, Claim, Unlock)
- Lead with benefit, not action
- Remove friction words (Buy, Pay, Submit)
- Add risk reversal when possible

## Copywriting Principles

### 1. Clarity Over Cleverness
- If the headline requires thought, rewrite it
- Use words a 12-year-old understands
- One idea per sentence

### 2. Specificity Builds Credibility
- "Save 10 hours/week" beats "Save time"
- "Used by 50,000+ teams" beats "Trusted by many"
- "Cut costs by 34%" beats "Reduce expenses"

### 3. Lead with Benefits
- Bad: "Our API supports 50+ integrations"
- Good: "Connect your entire tech stack in minutes—no developer required"

### 4. Address Objections Proactively
Common SaaS objections and responses:
- **"It's too expensive"** → Show ROI, offer free trial, highlight cost of inaction
- **"It's too complicated"** → Emphasize ease of use, onboarding support, templates
- **"I don't have time to switch"** → Offer migration services, done-for-you setup
- **"Will this work for my industry?"** → Provide industry-specific examples/case studies

## Section-by-Section Guide

### Hero Section
**Goal**: Capture attention and communicate core value in 5 seconds

**Checklist:**
- [ ] Headline communicates primary outcome/benefit
- [ ] Subheadline supports with context or secondary benefit
- [ ] CTA is clear, benefit-driven, and low-risk
- [ ] Visual supports the message (product screenshot or illustration)

### Features Section
**Goal**: Show how the product delivers the promised outcome

**Checklist:**
- [ ] Each feature has a benefit-driven headline
- [ ] Copy focuses on "what this means for you"
- [ ] Icons/graphics reinforce the message
- [ ] Features organized by importance or user journey

### Social Proof Section
**Goal**: Build trust and reduce perceived risk

**Elements:**
- Customer logos (familiarity bias)
- Testimonials (specific results > generic praise)
- Case studies (transformation stories)
- Stats ("Trusted by X companies" / "Y tasks automated")

### Pricing Section
**Goal**: Make choosing a plan effortless

**Best Practices:**
- Lead with most popular/middle tier
- Use plan names that describe the user (Starter, Professional, Enterprise)
- Highlight what's included, not just the price
- Add FAQ to address common pricing objections

### Final CTA Section
**Goal**: Remove last objections and drive action

**Formula:**
1. Restate primary benefit
2. Add urgency or scarcity
3. Reinforce risk reversal
4. Clear, prominent CTA

## Tone Guidelines by Vertical

### B2B SaaS
- Professional but approachable
- Data-driven (cite specific metrics)
- Focus on efficiency, ROI, and competitive advantage

### Developer Tools
- Technical but not jargon-heavy
- Show, don't just tell (code snippets, demos)
- Emphasize developer experience (DX)

### B2C/Freemium
- Conversational and friendly
- Emotional benefits over logic
- Social proof and FOMO work well

### Enterprise
- Security and compliance mentions
- Emphasis on support and reliability
- ROI calculators and case studies

## Output Format

When delivering landing page copy, use this structure:

```markdown
## [Section Name]

**Headline:** [Copy]
**Subheadline:** [Copy]
**Body:** [Copy]
**CTA:** [Copy]

---
```

## Common Mistakes to Avoid

1. **Focusing on features instead of outcomes**
2. **Using vague superlatives** ("best", "revolutionary", "innovative")
3. **Writing for everyone instead of a specific persona**
4. **Making the CTA too small or unclear**
5. **Ignoring mobile experience**
6. **Too much text before the first CTA**
7. **Not addressing the #1 objection**
